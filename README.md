## 태그속성 class, id
### class
* `<태그 class="값"></태그>`
* 2번이상 반복하는 이름인 경우
* 예) lnb, snb, container 등
### id
* `<태그 id="값"></태그>`
* 반복하면 안되는 단독 명칭인 경우
* 예) header, gnb, footer 등
## HTML 기초 문서 태그
* 블록태그 : w100%, 새로운 행, 크기가 큼
* 인라인태그 : 열(옆으로 나열), 크기는 내용만큼 인식
### 제목
* `h1~h6` : 블록태그, h1대제목, h6소제목, h레벨이 높을수록(1~3) 검색어로 인식될 확률 높음. SEO마케팅 목적.
### 내용
* `p` : 블록태그, 한줄~여러줄 단락형태의 내용일때
* `br` : 인라인태그, 줄바꿈할때 사용
* `em` : 인라인태그, 단순 문맥 강조, 주로 p 자식으로 사용(h에는 사용하지않음)
* `strong` : 인라인태그, 중요+심각성 강조, p 자식위주(h에는 사용하지않음)
* `blockquote` : 블록태그, 긴 인용문
* `q` : 인라인태그, 짧은 인용문
* `code` : 인라인태그, 컴퓨터 명령어를 화면에 표시할때
* `address` : 블록태그, footer안 주소 및 연락처, 해당 서비스 주소만 포함, address 안 블록 사용 주의(유효성에러)
### 레이아웃
* `div` : 블록태그, 그룹, 자식이 2개 이상이며 같은 방향일때, 블록과 인라인을 모두 자식으로 가질 수 있음
* `span` : 인라인태그, 그룹(자식이 2개 이상이며 같은 방향일때), 보조(디자인), 인라인만 자식으로 가질 수 있음, (주로 제목 h와 사용됨)
* `hr` : 블록태그, 경계구분용(디자인 테두리 용도 아님)
### 특수문자 태그
* `&copy;` : footer 저작권마크 ©
## 링크태그
* 필수구조 : `<a href="이동주소">클릭대상</a>`
* 링크 새창 열기 속성 : `<a target="_blank">클릭대상</a>`
### 링크태그 href 주소 작성법
* 절대경로 : `href="https://주소"` 실제 서비스 되는 웹주소
* 상대경로 : `href="./폴더 및 파일"` 같은 위치 또는 하위 위치
* 상대경로 : `href="../폴더 및 파일"` 상위 위치
### 바로가기링크(스크롤 이동하며 특정 위치로 가기)
1. `<a></a>` 태그 준비
2. 1번 링크 클릭시 이동하고 싶은 위치에 해당하는 태그에 `id="위치명"` 입력하기, 예) `<body id="top"></body>`
3. 1번 링크에 2번 아이디 연결하기 `<a href="#이동아이디명"></a>` **(#필수)**
* `#이름 <a href="#이동아이디명"></a>` : #옆에 이름을 쓰면 아이디란 의미
* `# <a href="#"></a>` : (연결파일이 준비되지 않은 경우) 임시링크인 경우 링크가 만들어지면 #표시 지워야 함 
* `<a href="#./index.html"></a>` = X
* `<a href="./index.html"></a>` = O

## background-image (CSS 배경이미지)
### `img` 태그가 아닌 background-image 처리하는 경우
* 형제, 자식, 자손 요소와 겹쳐져서 디자인 된 경우
* 예) 쇼핑몰 상품 썸네일, 히어로 배너, 광고배너 등
### 태그에 배경이미지 적용하는 작성순서
* `arl=""`는 없음
* `<태그 style="">`
* `<태그 style="background-image:;">`
* `<태그 style="background-image:url(경로);">`

## 유튜브 영상
* 원하는 영상 재생 -> 우클릭(소스코드복사)
### 유튜브 영상 속성
* `<iframe src-"https://youyube.com/embed/동영상이름">`
* `<iframe src-"https://youyube.com/embed/동영상이름?속성=값&속성=값">`
### 유튜브 영상 속성 종류
* `autoplay=1` : 재생함, 재생안함(0)
* `mute=1` : 음소거함, 음소거안함(0)
* `loop=1` : 반복재생함, 반복재생안함(0)
* `playlist=동영상이름` : loop=1일 경우 함께 사용(동영상 이름은 /embed/ 뒤에 있음.)
* `control=0` : 컨트롤숨김, 컨트롤보임(1)
## 목록태그 + 레이아웃(목록태그는 블록)
### 비순차 ul + li
* `<ul><li></li>*2</ul>`
* ul 비순차 그룹 li의 부모
* ul의 자식은 li만 올 수 있다. 자손은 모두 가능
### 순차 ol + li
* `<ol><li></li>*2</ol>`
* ol 순차 그룹 li의 부모
* ol의 자식은 li만 올 수 있다. 자손은 모두 가능
## 비순차목록
* `dl`그룹, `dt`제목, `dd`내용
* `dt-dd`는 h4~h6 정도로 구성된 제목과 내용이 연속적으로 2개 이상 구성되어있을 경우 사용한다.
* `dl`의 자식은 `dt`, `dd`만 올 수 있음.
* `dt-dd` 형제는 `dt-dd`만 가능함.
## Form 태그
### `<form action="#" method="#"></form>`
* action : 사용자가 입력 또는 선택한 정보를 전송하는 주소
    * 네이버 로그인 중이라면 아이디, 비밀번호를 네이버 서버로 전송해서 유효성 체크 
* method : action에 정보를 전송할때 어떤 방식으로 전송할 것인지 결정하는 방식 **get, post**
    * get : 주소창에 입력정보 공개, 보안(X)
    * post : 입력정보 비공개, 보안(O)
### `<imput>` 입력 및 선택양식
* `<imput type="" name="" value="" id="" class="">`
* type : 용도에 따른 종류선택(ex)비번=password
* name 
    * 입력양식(text, password 등) : 서버 전송 시 데이터 구분을 위한 개별이름
    * 선택양식(select, checkbox, radio) : 선택 요소 2개 이상을 묶는 그룹명(숫자X)
* value(가능한 쓰는게 좋다~)
    * 입력양식 : 처음부터 입력되어있는 초기값(수량 1)
    * 선택양식 : 데이터 구분용(name 의미동일)
* id : css or js 데이터 구분명(name 동일설정가능)
* class : 반복 디자인 사용 시 설정하는 반복이름
### button
* `<button type="종류"></button>`
* `name, value` 속성 사용 안함. `id+class`는 필요에 따라 사용함
* button
    * submit, reset을 제외한 모든 범용적 기능
    * 사용자의 입력(input), 선택(select, checkbox, radio) 요소가 없는 경우 단순 버튼의 기능만 활용할 때 form 없이 버튼만 사용할 수있다.(이전, 다음, 재생, 일시정지 등)
    * 페이지 내에서 이전/다음 이동, 재생, 일시정지, 정지 등 다양한 기능 활용
    * 서버 전송이 아닌 조건에 따른 다른 창 띄우기(우편번호찾기, 본인인증 등)
* submit
    * form action, method 방식 서버에 전송하는 데이터
    * (예) 아이디, 비번 입력 후 서버 전송해서 유효성 검사
    * (예) 상품을 장바구니(서버)에 담기
    * (예) 장바구니에 담은 상품을 구입해서 내 구입 목록(서버)에 저장하기
* reset
    * 기존 데이터를 현재 페이지에서 제거, 취소할 경우
